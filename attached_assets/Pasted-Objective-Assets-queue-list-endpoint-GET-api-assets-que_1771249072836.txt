Objective
- Assets queue list endpoint (GET /api/assets-queue) must meet p95 < 2 seconds under typical load.
- List view can return minimal fields only. Expanded row details may load in 1–3 seconds with a “Loading details…” state.
- List is already paginated.

Critical requirement
- Keep Snowflake off the list endpoint’s critical path: no per-item Snowflake calls, and no Snowflake-dependent filters/sorts on list load.

Option A (preferred): Materialize into Postgres

- Graceful degradation: list must load even if Snowflake is down; expanded details may show partial data.
