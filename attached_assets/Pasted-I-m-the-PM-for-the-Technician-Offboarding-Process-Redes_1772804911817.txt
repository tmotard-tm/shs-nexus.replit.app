Change 3: Add is_tlt column
Add is_tlt (boolean, default false) to queue_items. Set it at sync time based on the technician's job title from Snowflake.

Change 4: Propagate is_byov / vehicle_type to all 5 task rows

Update the sync logic in syncTermedTechs() so that BYOV detection runs once per tech and the result is applied to all 5 task rows created for that tech.

Acceptance: All 5 task rows for a BYOV tech have is_byov = true and vehicle_type = "byov". Backfill applied to existing data.

Change 5: Conditional Phone Recovery task creation
Currently, the sync creates a Phone Recovery task for every tech regardless of whether they have a company phone. This creates noise in Monica's queue.

Question: Is there a field in the Snowflake data (term roster or separation details) that indicates whether a tech has a company-issued phone? For example, a phone number field, a device assignment field, or a flag?

Acceptance: Phone Recovery tasks only created for techs with company phones (or flagged as blocked with explanation).

Implement a mechanism to move queue_items rows with status = 'completed' or status = 'cancelled' that are older than 90 days to an archive_queue_items table (same schema).

Run this as a scheduled job (e.g., weekly) or as part of the existing sync scheduler.

Acceptance: Archival job exists. Old completed/cancelled rows moved to archive table. Active queries only hit the main table. Archive table queryable for historical analysis.
