3) Forms → Task Router (server-side)

All forms submit to one endpoint; the server creates 1..N tasks and returns them.

Endpoint

POST /api/forms/{key}/submit
Body: { ...form fields... }
→ Validates → Creates tasks via router →
Response:
{
  "task_ids": [123,124,...],
  "created": [
    {"id":123,"department":"FLEET","type":"Stop Truck Replenishment"},
    {"id":124,"department":"PARTS_RECOVERY","type":"Full Truck Count & Return"}
  ]
}

C) Onboarding (key: onboarding)

Spawns

NTAO → Create Tech Record & Access (ntao_create_user_v1, priority:1)

ASSETS → Provision Assets (assets_provision_v1, priority:2)

(if applicable) PARTS_RECOVERY → Seed/Enable Truck Stock (parts_seed_stock_v1, priority:3)

D) Offboarding (key: offboarding)

Spawns

One login across all queues.

Each form submission spawns tasks in the correct queues per rules above.

Queues filter by department; agents only see what they’re permitted.

Pick up → /tasks/:id/work with correct checklist.

Deep links for forms and tasks work with login redirect.

Offboarding spawns all required tasks across Fleet, Assets, Inventory, NTAO.
