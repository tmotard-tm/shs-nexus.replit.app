Sprint 0: Enable Snowflake Sync for Tool Audit Notifications
I'm the PM for the Technician Offboarding Process Redesign. This is Sprint 0 of the Tool Audit Notification feature.

Context
We're implementing automated notifications to separating technicians. The goal is to increase voluntary tool audit completion from ~20% to a much higher rate. Notifications will trigger when Crystal's Snowflake table syncs new separation records.

Key Constraint
DO NOT modify Luca's Tool Audit form. The form serves multiple purposes (onboarding, offboarding, regular audits). This feature only adds Nexus sync logic and email notification.

What You're Building
Nexus needs to poll Crystal's Snowflake separation table and detect new records. When a new separation record appears, Nexus should:

Create offboarding tasks for Fleet/Tools/Inventory teams (if not already doing this)
Store the separation data locally for reference
Snowflake Table Schema
Crystal's separation intake table has these fields:

Column	Type	Nullable	Notes
ID	NUMBER(38,0)	N	Primary key, auto-increment
LDAP_ID	VARCHAR(50)	N	Tech identifier (use to pre-fill form)
TECHNICIAN_NAME	VARCHAR(200)	Y	Display name
EMPLID	VARCHAR(20)	Y	Employee ID
PLANNING_AREA	VARCHAR(20)	Y	District code
PLANNING_AREA_NAME	VARCHAR(200)	Y	District name
SEPARATION_CATEGORY	VARCHAR(100)	Y	Vol/Invol/JA
LAST_DAY	DATE	Y	Last day of employment
EFFECTIVE_SEPARATION_DATE	DATE	Y	Separation effective date
CONTACT_NUMBER	VARCHAR(50)	Y	Phone (for future SMS)
TRUCK_NUMBER	VARCHAR(50)	Y	Vehicle number
FLEET_PICKUP_ADDRESS	VARCHAR(500)	Y	Pickup location
PERSONAL_EMAIL	VARCHAR(200)	Y	Key field for notification
CONTACTED_BY	VARCHAR(200)	Y	Who completed separation call
NOTES	VARCHAR(4000)	Y	Free text notes
CREATED_AT	TIMESTAMP_NTZ(9)	Y	Record creation time
UPDATED_AT	TIMESTAMP_NTZ(9)	Y	Last update time
Implementation Steps
Snowflake Connection

Ensure Snowflake credentials are configured (check Replit Secrets for SNOWFLAKE_* keys)
Test query access to Crystal's table
Sync Service

Extend existing server/snowflake-sync-service.ts (do NOT create new service)
Poll for new records (check CREATED_AT > last sync time)
On new record: create offboarding tasks for Fleet, Tools, Inventory queues
Store separation data: Add separation record fields to queue_items metadata or offboarding record (use existing data model, do not create new table for Sprint 0)
Sync Frequency

Start with every 5 minutes
Can adjust based on requirements
Note: If snowflake-sync-service.ts already exists with partial sync logic, extend it. Do not duplicate sync infrastructure.

Acceptance Criteria
 Nexus can query Crystal's Snowflake separation table
 New separation records trigger offboarding task creation
 Sync runs automatically on schedule
 Logs show successful syncs and any errors
