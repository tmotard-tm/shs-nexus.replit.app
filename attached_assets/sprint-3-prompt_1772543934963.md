## Sprint 3: Phone Recovery Queue Dashboard

I'm the PM for the Technician Offboarding Process Redesign. This is Sprint 3 of the Phone Recovery Queue feature. Sprints 0-2 are complete: schema is in place, and both the contact logging components (Sprint 1) and reprovisioning checklist component (Sprint 2) are built.

### Context

The Stage 1 and Stage 2 form components are built as reusable React components in `client/src/components/phone-recovery/`. Utility functions `deriveRecoveryStatus()`, `deriveReprovisioningStatus()`, and `isEscalated()` are available.

This sprint assembles everything into the main Phone Recovery Queue page: a dashboard-first view where clicking any task opens a slide-out detail panel with the appropriate form.

The goal: the manager should be able to open this view and immediately know how many phones are in each stage, which cases need attention, and how many devices are ready for new hires. Any team member should be able to click a task and take action directly from the detail panel.

### What You're Building

A single dashboard page at a new route (e.g., `/phone-recovery`). This is the only page for the Phone Recovery Queue feature. No separate Stage 1 or Stage 2 pages.

### Component 1: Pipeline Summary Cards (row of 6)

Six cards in a horizontal row at the top of the page:

| Card | Label | Filter Logic | Left Border Color |
|------|-------|-------------|-------------------|
| 1 | New | Recovery stage, phoneContactHistory is empty | Blue (#003366) |
| 2 | In Contact | Recovery stage, phoneContactHistory has entries, phoneDateReceived is null | Amber |
| 3 | In Transit | Recovery stage, phoneShippingLabelSent = true, phoneDateReceived is null | Blue (#003366) |
| 4 | Reprovisioning | Reprovisioning stage, phoneServiceReinstated = false | Purple |
| 5 | Ready | Reprovisioning stage, phoneServiceReinstated = true, phoneAssignedToNewHire is null | Emerald |
| 6 | Assigned | phoneAssignedToNewHire is not null | Emerald |

Each card shows: label text, large count number, color-coded left border.

**Behavior:**
- Clicking a card filters the task table below to show only tasks matching that status group
- Clicking the same card again clears the filter
- Active (selected) card shows a ring highlight and small "Filtered" text indicator
- Hover state: subtle lift + shadow

**Status grouping for filtering:**
- "In Contact" maps to status = Contact Attempted
- "In Transit" maps to statuses Label Sent + In Transit
- "Reprovisioning" maps to statuses Received + Inspecting + Wiping + Reprovisioning

### Component 2: Aging Alerts (2/3 width) + Device Inventory (1/3 width)

Place these side by side on the second row.

**Aging Alerts card (left, 2/3 width):**
- Header: amber warning icon (AlertTriangle from lucide-react) + "Aging Alerts" bold + "Tasks open >5 days" subtext
- Grid of up to 4 alert mini-cards. Each shows: technician name, status badge (color-coded), days open count (right-aligned, bold)
- Escalated tasks: red left border + red AlertTriangle icon
- Non-escalated aging tasks: amber left border + amber background tint
- Each mini-card is clickable: opens the detail panel for that task
- If more than 4 aging tasks exist, show a "+N more" indicator

**Aging calculation:**
- Recovery stage: compare current date against `created_at`
- Reprovisioning stage: compare current date against `phoneDateReceived`
- Aging = open > 5 business days

**Device Inventory card (right, 1/3 width):**
- Three line items with icons and counts:
  - CheckCircle icon (emerald): "Ready for Deployment" -- count of tasks where phoneServiceReinstated = true and phoneAssignedToNewHire is null
  - RefreshCw icon (purple): "In Reprovisioning" -- count of tasks in reprovisioning stage that are not yet ready
  - XCircle icon (gray): "Written Off" -- count of tasks where phoneWrittenOff = true
- Below the counts: horizontal stacked bar showing proportions (emerald / purple / gray segments)

### Component 3: Filter Bar

A white card with four filters in a row:

1. **Stage** dropdown: All Stages, Recovery, Reprovisioning
2. **Status** dropdown: All Statuses, New, Contact Attempted, Label Sent, In Transit, Received, Inspecting, Wiping, Reprovisioning, Ready for Deployment, Assigned
3. **Team Member** dropdown: All Members, then each unique assignedTo value
4. **Separation Date** range: two date inputs (From -- To) with calendar icons (Calendar from lucide-react)

Right side of the filter bar:
- "Clear all filters" link/button (only visible when any filter is active, including pipeline card selection). Clears all dropdowns, date range, AND deselects any active pipeline card.
- "Showing X of Y tasks" count text

All filters work together with AND logic. Pipeline card selection counts as a filter.

### Component 4: Task Table

Full-width table with these columns:

| Column | Content | Notes |
|--------|---------|-------|
| Technician | Name only | Do NOT show device model or phone number |
| Separation Date | Formatted date | |
| Stage | Pill badge: "Recovery" (blue bg) or "Reprovisioning" (purple bg) | Do NOT show "Stage 1" or "Stage 2" |
| Status | Color-coded pill badge | Blue = New, Label Sent, In Transit. Amber = Contact Attempted. Emerald = Received, Ready for Deployment, Assigned. Purple = Inspecting, Wiping, Reprovisioning |
| Days Open | Number | Red text if > 14 days, amber text if > 5 days, default otherwise |
| Assigned To | Team member name | |
| Alert | Independent indicators | Red AlertTriangle + "Escalation" if isEscalated(). Amber Clock + "Aging" if aging > 5 days. Both can appear on same row. Dash if neither |

**Row behavior:**
- Hover: subtle background highlight
- Click: opens the slide-out detail panel for that task
- Escalated rows: subtle red background tint (red-50)
- Aging-only rows: subtle amber background tint (amber-50)
- If both escalated and aging: use red tint (escalation takes priority)

**Default sort:** Days open, descending (longest first).
**Column sort:** Column headers are clickable to toggle ascending/descending sort.

**Table column headers:** Uppercase, tracking-wider, text-xs, text-gray-500.

### Component 5: Slide-Out Detail Panel

Triggered by clicking any task row or aging alert mini-card. Slides in from the right side. Dark backdrop overlay behind the panel.

**Panel width:** ~480px (or similar comfortable width for forms).

**Header section:**
- Left: Technician name (text-lg, bold) + status badge (color-coded pill) + escalation badge (red, only if isEscalated())
- Right: Close button (X icon)

**Info grid (2x2):**
| Label | Value |
|-------|-------|
| Separation Date | Formatted date |
| Phone Number | phoneNumber |
| Days Open | Count with color coding |
| Assigned To | Team member name |

**Tabs (two tabs):**

**Tab 1: "Contact History"**
- Import and render the `ContactHistoryTimeline` component from Sprint 1
- Pass the task's `phoneContactHistory` array
- Shows vertical timeline of all attempts: method icon, method name, date, outcome (color-coded), notes
- Shows shipping label status with tracking number if sent

**Tab 2: Dynamic based on stage**
- If task stage is "initiation" (Recovery): Label tab "Log Contact". Import and render `ContactLogForm` component from Sprint 1. Include the "Mark Received" button below the form.
- If task stage is "reprovisioning": Label tab "Reprovisioning". Import and render `ReprovisioningChecklist` component from Sprint 2.

**Footer:**
- Left: "Close" text button
- Right: "Save Changes" primary button (blue #003366 background, white text)

**Behavior:**
- After saving (contact log or reprovisioning step), the task list and pipeline cards should refresh to reflect the updated state
- Use TanStack Query cache invalidation to refresh data after mutations
- Panel can be closed by clicking the X button, the Close button, or clicking the dark backdrop

### Visual Design Specifications

**Color palette:**
- Primary: #003366 (header accents, primary buttons, focus rings, active card rings)
- Status blues: blue-100 bg / blue-700 text for New, Label Sent, In Transit
- Status amber: amber-100 bg / amber-700 text for Contact Attempted
- Status emerald: emerald-100 bg / emerald-700 text for Received, Ready, Assigned
- Status purple: purple-100 bg / purple-700 text for Inspecting, Wiping, Reprovisioning
- Escalation red: red-100 bg / red-700 text for alerts, red-50 for row tint
- Aging amber: amber-50 for row tint

**Typography:**
- Table cells: text-sm
- Card content: text-base
- Section headers: text-lg or text-xl
- Table column headers: text-xs, uppercase, tracking-wider, text-gray-500

**Spacing:**
- Cards: p-4 or p-6
- Grid gaps: gap-4
- Page padding: p-6

**Interactive states:**
- Cards: hover:shadow-md, hover:-translate-y-0.5, transition-all
- Table rows: hover:bg-gray-50
- Inputs: focus ring with primary blue
- Buttons: hover:opacity-90 for primary, hover:bg-gray-100 for secondary

**Background:** Page bg-gray-100, cards bg-white with border border-gray-200 and shadow-sm.
