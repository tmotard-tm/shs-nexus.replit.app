# Changelog — 2026-02-13

## Summary
Consolidated Assets Queue with vehicle disposition integration, whitelist email behavior changes, Communication Hub navigation update, and offboarding communication compliance updates.

---

## Changes

### Communication Hub — Whitelist Mode Behavior Change
- **What**: Whitelist mode now sends emails TO all whitelisted addresses (instead of only sending to the original recipient if they were on the whitelist)
- **Why**: Enables testing by routing all test emails to whitelisted addresses regardless of the original intended recipient
- **Details**: Subject line prefixed with `[TEST - Original recipient: user@example.com]` so testers know who the email was originally for
- **Files**: `server/communication-service.ts`

### Offboarding Communication — Legal Compliance
- **What**: Removed "payroll adjustments/deductions" language from all user-facing communication templates
- **Why**: Legal compliance — communications should not reference payroll actions
- **Files**: Communication templates (embedded defaults)

### Communication Hub — Navigation Update
- **What**: Moved Communication Hub link from standalone sidebar item into the Activity section
- **Why**: Better organization; Communication Hub is an activity/audit tool, not a top-level page
- **Details**: Only visible to developers (super administrators) with proper role permissions
- **Files**: Sidebar navigation components, role permissions configuration

### Vehicle Disposition Integration
- **What**: Replaced manual Vehicle Routing radio buttons (PMF / Pep Boys / Transfer / Pending) with read-only Disposition field sourced from `vehicle_nexus_data.postOffboardedStatus`
- **Why**: Vehicle disposition is now set on the Weekly Offboarding page, making the Assets Queue a consumer of that data rather than a place to set it
- **Details**:
  - Assets Queue table column renamed from "Routing" to "Disposition"
  - Batch-fetched vehicle nexus data for efficient display in table rows
  - Detail view shows disposition as read-only badge with color coding
  - Section heading updated from "Contact & Routing" to "Contact & Disposition"
  - Removed unused `RoutingDestination` type and `getRouting()` function
- **Files**: `client/src/components/assets-queue/AssetsRecoveryQueue.tsx`, `client/src/components/assets-queue/AssetsTaskDetailView.tsx`

### Housekeeping
- Deleted 7 unused screenshot images from `attached_assets/` folder
- Retained only the Sears service van image referenced in code
