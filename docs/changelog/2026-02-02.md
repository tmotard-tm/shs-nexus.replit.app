## Bug Fixes

### Critical: Phase 2 Trigger Blocked by workflowStep Check
- **Problem**: `triggerNextWorkflowStep()` had early return `if (!workflowStep)` that blocked Day 0 task detection
- **Cause**: Day 0 tasks don't always have `workflowStep` set
- **Fix**: Moved Day 0 task detection BEFORE `workflowStep` requirement
- **Files**: `server/storage.ts` (both MemStorage and DatabaseStorage classes)

### Routing Decision Not Persisting
- **Problem**: `/api/tools-queue/:id/assign` wasn't saving `fleetRoutingDecision` or clearing `blockedActions`
- **Fix**: Updated endpoint to save routing decision and clear blocked actions when routing is provided
- **Files**: `server/routes.ts`

### Phase 2 Task Titles Showing "undefined"
- **Problem**: Phase 2 task titles displayed "undefined" instead of technician name
- **Cause**: `createPhase2FleetTasks()` was looking for `triggerData.vehicle?.vehicleNumber` which wasn't set
- **Fix**: Added fallback chain to extract techName, vehicleNumber, employeeId from multiple possible locations
- **Files**: `server/storage.ts`
