# Changelog: 2026-02-05

## Communication Hub MVP

### What We Built
1. **Centralized Template Management**
   - New page at `/communication-hub` (Developer-only access)
   - Three tabs: Templates, Whitelist, History
   - Edit email/SMS template content with live preview
   - Toggle between simulated/whitelisted/live modes per template

2. **Mode-Aware Send Logic**
   - `simulated`: Logs message without actually sending (default for safety)
   - `whitelisted`: Only sends if recipient is in the whitelist
   - `live`: Sends to the actual recipient

3. **Whitelist Manager**
   - Add/remove test emails and phone numbers
   - Whitelisted mode uses this list for safe testing
   - Search and filter whitelist entries

4. **Send History**
   - View all communication logs with filtering
   - Shows status (sent/simulated/failed), recipient, timestamp
   - Filter by status for easy debugging

5. **Developer-Only Access Control**
   - All `/api/communication/*` routes require developer role
   - `requireDeveloperRole()` helper validates session and role
   - Frontend uses page-registry permissions for UI visibility

## Sprint 0: Snowflake Separation Sync for Offboarding Automation

### What We Built
1. **New Separation Detection**
   - `getSeparationsSinceTimestamp()` method queries HR table for records created after last sync
   - Uses `CREATED_AT > ?` filter to detect only new entries
   - Returns full separation details including contact info, truck number, fleet pickup address

2. **Automatic Offboarding Task Creation**
   - `syncNewSeparations()` creates Day 0 tasks for Tools, Fleet, and Inventory queues
   - Includes technician info, vehicle details, and HR notes in task data
   - Deduplication: Skips if existing tasks found within 45-day window

3. **5-Minute Polling Scheduler**
   - Added `checkAndRunSeparationPoll()` to sync-scheduler.ts
   - Polls every 5 minutes for new separation records
   - Logs results and tracks last poll timestamp

4. **Manual Trigger Endpoint**
   - `POST /api/snowflake/sync/separations` for testing and manual sync
   - Developer/Admin role required

### Files Modified
- `server/snowflake-sync-service.ts` - Added getSeparationsSinceTimestamp(), syncNewSeparations()
- `server/sync-scheduler.ts` - Added 5-minute separation poll, triggerSeparationPoll()
- `server/routes.ts` - Added POST /api/snowflake/sync/separations endpoint

### Technical Details
- **HR Table**: `PRD_TECH_RECRUITMENT.FLEET_DETAILS.SEPARATION_FLEET_DETAILS`
- **Sync Log Type**: `separation_poll` (tracks last successful poll)
- **Default Lookback**: 24 hours if no previous sync log exists
- **Tasks Created**: 3 per separation (Tools, Fleet, Inventory)

### Acceptance Criteria Met
- ✅ Nexus can query Crystal's Snowflake separation table
- ✅ New separation records trigger offboarding task creation
- ✅ Sync runs automatically on schedule (every 5 minutes)
- ✅ Logs show successful syncs and any errors
