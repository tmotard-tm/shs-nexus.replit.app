# Changelog: 2026-02-11

## Assets Queue Enhancements

### 1. Dedicated Assets Recovery Queue View
- Built `AssetsRecoveryQueue.tsx` component with table-based layout matching Tools Queue pattern
- Table features: search, filters (status/vehicle/district), sortable columns, pagination, expandable inline rows
- Expandable rows include: contact details sidebar, recovery task checkboxes with auto-save, vehicle routing radio buttons, carrier dropdown, notes section, quick actions

### 2. NTAO-Style Collapsible Card Header
- Wrapped AssetsRecoveryQueue with collapsible Card header (green color bar, icon, item count, status badges, expand/collapse toggle)
- Matches visual pattern used by all other department queues in unified Task Queue page

### 3. Fleet Separation Source Detection & Filtering
- Added `getItemSource()` function to classify items as "fleet_separation" (from Snowflake SEPARATION_FLEET_DETAILS) or "manual" (created in-app)
- Source values detected: `snowflake_sync`, `hr_separation`, `hr_separation_sync`
- Added subtle italicized gray text labels showing "Fleet Separation" or "Manual" below enterprise ID in each row
- Added "Include Manual" filter toggle (defaults to off) to hide test/manual items

### 4. Split Pick Up / Assign Button
- Split single "Pick Up / Assign" button into two distinct actions:
  - **Pick Up**: One-click auto-assigns item to current logged-in user via `assignMutation`
  - **Assign**: Opens existing `PickUpRequestDialog` for selecting a different user
- Added `onQuickPickUp` prop to `ExpandedRowDetails` component

### Files Modified
- `client/src/components/assets-queue/AssetsRecoveryQueue.tsx` - New component with all features above
- `client/src/pages/queue-management.tsx` - Integration of Assets Queue card header
